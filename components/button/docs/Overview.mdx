# Variants
## Primary Button

Primary button calls attention to an action or highlights the most important call to action in a section. 

**Example : Save, Confirm, Done**

<Documenter component={props.components}>

```preview
return (
  <Button type="primary">Button Label</Button>
)

```

</Documenter>

## Secondary Button

Secondary buttons are used to highlight secondary actions. Remember to maintain visual hierarchy by balancing weights and background colours. 

<Documenter component={props.components}>

```preview
return (
  <Button type="secondary">Button Label</Button>
)

```

</Documenter>

## Tertiary Button

Tertiary buttons will indicate the least significant actions on the page. Pair them with primary and secondary buttons when there are multiple calls to action. 

<Documenter component={props.components}>

```preview
return (
  <Button type="tertiary">Button Label</Button>
)

```

</Documenter>

## Warning Button

Warning buttons are used to get a final confirmation from a user in the context of a destructive or negative action. These are often used in confirmation modals. 

**Example :  Delete, Remove, Decline**

<Documenter component={props.components}>

```preview
return (
  <Button type="warning">Button Label</Button>
)

```

</Documenter>


## Icon only Button

Icon Only buttons are used when pairing an Icon with an action to control the interaction. Use them with Tooltips to add clarity to the action. 

**Example :  More**

<Documenter component={props.components}>

```preview
return (
  <Button icon={EllipsisVSolid} />
)

```

</Documenter>
